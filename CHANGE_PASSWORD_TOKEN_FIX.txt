PROTRADE CHANGE PASSWORD TOKEN FIX

Problem fixed:
The Settings page called /api/auth/change-password without an Authorization
Bearer token because auth.js excluded every /api/auth/* request from its
automatic token injection.

Fix:
- auth.js now excludes only public login and register requests.
- Protected auth routes such as /auth/change-password receive the JWT.
- settings.html also sends the Bearer token explicitly as a safeguard.
